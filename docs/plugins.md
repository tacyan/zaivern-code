# プラグイン開発ガイド

Zaivern Code の機能は、**シェルスクリプトが書ければ誰でも増やせます**。
Rust を書く必要はなく、再ビルドも不要です。フォルダを 1 つ置くだけで、
コマンド・パネル・自動実行フック・テーマ・スニペットを追加できます。

標準プラグインもまったく同じ仕組みの上に乗っています。つまり
**標準機能を読めば、そのまま自作の手本になります**（`~/.zaivern/plugins/` を覗いてください）。

## 3 分で作る

```bash
zai plugin new hello
```

`~/.zaivern/plugins/hello/` に雛形一式が生成されます。
サイドバーの 🔌 タブからも ➕ ボタンで同じことができます。

生成された `plugin.toml` を開いて、こう書き換えてみてください。

```toml
[plugin]
name = "hello"
version = "0.1.0"
description = "はじめてのプラグイン"
api = 2

[[command]]
title = "今日の日付を挿入"
run = "date '+%Y-%m-%d'"
input = "none"
output = "insert"
keybind = "cmd+alt+d"
```

サイドバーの ⟳ で再読み込みすると、もうコマンドパレット（`Cmd+Shift+P`）に
「hello: 今日の日付を挿入」が並び、`Cmd+Alt+D` で日付が挿入されます。

これだけです。**やっていることは「コマンドを実行して、標準出力をどこかへ流す」だけ**で、
あとはその応用でしかありません。

## プラグインの構造

```
~/.zaivern/plugins/<名前>/
├── plugin.toml          必須。これが無いフォルダは無視されます
├── scripts/             任意。シェルスクリプト置き場（名前は自由）
│   └── run.sh
├── themes/              任意。カラーテーマ JSON
│   └── my-theme.json
└── snippets/            任意。スニペット JSON
    └── rust.json
```

- 名前に使えるのは `a-z` `0-9` `_` `-` のみ（最大 64 文字）です。
- 1 階層だけ走査します。フォルダの入れ子は認識されません。
- マニフェストが壊れていても他のプラグインは巻き添えになりません。
  壊れたものは一覧に ⚠ 付きで残り、理由が表示されます。

## マニフェスト (`plugin.toml`)

```toml
[plugin]
name = "example"          # 必須。フォルダ名と揃えるのが無難
version = "0.1.0"
author = "あなたの名前"
description = "何をするプラグインか"
api = 2                   # 省略時は 1（v1 の書き方もそのまま動きます）

[[command]]  ...          # 手で実行する機能
[[panel]]    ...          # サイドバーに出す表示
[[hook]]     ...          # 自動で走る処理
[[setting]]  ...          # ユーザーに設定させる値
[[theme]]    ...          # カラーテーマ
[[snippet]]  ...          # スニペット
```

`api = 2` を書かなくても動きますが、`[[panel]]` `[[hook]]` `[[setting]]` と
新しい出力先を使うなら `api = 2` を明記してください。

## コマンド

```toml
[[command]]
id = "fmt"                # 省略可。省略時は title から自動生成
title = "JSON を整形"      # UI に出る名前
icon = "✨"
run = "python3 -m json.tool"
input = "file"            # none | selection | file
output = "replace"        # 下表を参照
langs = ["json"]          # 空 or 省略なら全言語で出る
keybind = "cmd+alt+f"
on_save = true            # 保存時に自動実行（フォーマッタ向け）
timeout_secs = 30         # 1〜600
```

### `input` — スクリプトの標準入力に何を渡すか

| 値 | 渡すもの |
|---|---|
| `none` | 何も渡さない（既定） |
| `selection` | 選択範囲。未選択ならバッファ全体 |
| `file` | 現在のバッファ全体 |

### `output` — 標準出力をどこへ流すか

| 値 | 挙動 |
|---|---|
| `replace` | 入力に使った範囲を置き換える（整形・変換向け） |
| `insert` | カーソル位置に挿入する |
| `new_tab` | 新しいタブを開いて表示する |
| `notify` | トーストで通知する |
| `silent` | 捨てる（副作用だけが目的のとき） |
| `agent_prompt` | エージェントの入力欄へ差し込む（送信はしない） |
| `panel` | 指定した `panel` へ表示する |
| `actions` | **JSON で書いた指示としてアプリを操作する**（後述） |

`output = "panel"` のときは、出力先を `panel = "パネルID"` で指定します。

> **安全装置**: `replace` は、スクリプトが走っている間にあなたがバッファを
> 編集していた場合、結果を捨てます。書きかけの文章が消えることはありません。

### `on_save` の制約

`on_save = true` は `input = "file"` と `output = "replace"` の組み合わせでのみ有効です。
他の組み合わせだとマニフェストエラーになります（保存時に予期しない動作をさせないためです）。

## パネル

サイドバーに独自の表示領域を持てます。

```toml
[[panel]]
id = "tasks"              # 必須。プラグイン内で一意
title = "課題"
icon = "📋"
run = "sh scripts/list.sh"   # 空にすると、アクション経由でのみ更新
refresh = "on_open"       # manual | on_open | interval
interval_secs = 30        # refresh = "interval" のときのみ
format = "markdown"       # text | markdown
```

`format = "markdown"` にすると見出し・箇条書き・リンクが描画されます。
一覧やレポートを出すならこちらが読みやすくなります。

パネルの中身は次の 3 通りで更新できます。

1. `refresh` の設定に従って自動で `run` を実行する
2. `output = "panel"` のコマンドを実行する
3. 任意のスクリプトから `set_panel` アクションを流す

## フック（自動実行）

```toml
[[hook]]
event = "file_save"
run = "sh scripts/on-save.sh"
output = "silent"         # silent | notify | actions | panel
timeout_secs = 30
```

| `event` | 発火するタイミング |
|---|---|
| `startup` | 起動直後に 1 回 |
| `file_open` | ファイルを開いたとき |
| `file_save` | 保存に成功したとき |
| `agent_finish` | エージェントの作業が終わったとき |
| `agent_attention` | エージェントが承認待ちで止まったとき |
| `git_change` | git の状態（ブランチ・変更ファイル）が変わったとき |
| `interval` | 一定間隔ごと（`interval_secs` は 5 秒以上） |

どのイベントで呼ばれたかは環境変数 `ZV_EVENT` で判別できます。

> フックはバックグラウンドで走ります。重い処理を `file_save` に置くと
> 保存のたびに走るので、`interval` に逃がすか、スクリプト側で早期 return してください。

## 設定項目

ユーザーに値を入れてもらう欄を宣言できます。

```toml
[[setting]]
key = "host"
type = "string"           # string | bool | int
default = ""
label = "接続先ホスト"
secret = false            # true なら UI で伏せ字にする
```

設定された値は、スクリプトへ `ZV_CFG_<キーを大文字にしたもの>` として渡ります。
上の例なら `ZV_CFG_HOST` です。

ユーザー側は UI から編集できるほか、`~/.zaivern/config.toml` に直接書けます。

```toml
[plugins.settings.remote-host]
host = "user@example.com"
remote_path = "/home/user/work"
```

プロジェクト単位で上書きしたいときは、ワークスペース直下の `.zaivern.toml` に
同じ形式で書いてください。

## アクション — アプリを操作する

ここがこの仕組みの中心です。`output = "actions"` にすると、
**標準出力の各行を「アプリへの指示」として解釈します**（JSON Lines）。

```sh
#!/bin/sh
echo '{"action":"notify","message":"処理が終わりました"}'
echo '{"action":"open_file","path":"src/main.rs","line":42}'
```

これでスクリプトから、通知を出し、ファイルを開き、エージェントに話しかけ、
ターミナルを起こす、といったことが自由にできます。

### 使えるアクション

| アクション | 指定できる項目 | 動き |
|---|---|---|
| `open_file` | `path`, `line` | ファイルを開いて該当行へ飛ぶ |
| `notify` | `message`, `level` | トースト通知（`level` = `info` / `warn` / `error`） |
| `insert_text` | `text` | カーソル位置に挿入 |
| `replace_buffer` | `text` | バッファ全体を置き換え |
| `new_tab` | `title`, `text` | 新規タブを開いて表示 |
| `agent_prompt` | `agent`, `text`, `submit` | エージェント入力欄へ差し込む |
| `run_terminal` | `command`, `cwd` | ターミナルでコマンドを実行 |
| `open_url` | `url` | 既定のブラウザで開く |
| `set_panel` | `panel`, `text` | パネルの内容を差し替え |
| `set_status` | `text` | ステータスバーに表示 |
| `refresh_files` | — | ファイルツリーを再読み込み |
| `set_setting` | `key`, `value` | 自分の設定値を保存する |

`agent_prompt` の `submit` は既定が `false` です。
**`true` にしない限り勝手に送信はされません** — 内容を確認してから自分で Enter を押せます。

解釈できない行は無視されるだけで、プラグインは止まりません。
`echo` のデバッグ出力が混ざっていても壊れません。

## 環境変数

スクリプトには次の値が渡ります。

| 変数 | 中身 |
|---|---|
| `ZV_FILE` | 現在のファイルの絶対パス（未保存なら空） |
| `ZV_LANG` | 言語 ID（`rust` / `json` など） |
| `ZV_WORKSPACE` | ワークスペースのルート |
| `ZV_PLUGIN_DIR` | 自分のプラグインフォルダ |
| `ZV_PLUGIN_DATA` | 自分用の永続データ置き場（自動作成される） |
| `ZV_SELECTION` | 選択中のテキスト（未選択なら空） |
| `ZV_LINE` / `ZV_COLUMN` | カーソル位置（1 始まり） |
| `ZV_AGENT` | アクティブなエージェント名 |
| `ZV_EVENT` | フックのイベント名（コマンド実行時は空） |
| `ZV_GIT_BRANCH` | 現在のブランチ名（git 管理外なら空） |
| `ZV_BIN` | `zai` バイナリの絶対パス |
| `ZV_API` | プラグイン API のバージョン（`2`） |
| `ZV_CFG_<KEY>` | `[[setting]]` の現在値 |

作業ディレクトリはワークスペースのルートです。
スクリプトを相対パスで書くときは `"$ZV_PLUGIN_DIR/scripts/run.sh"` を使ってください。

**状態を保存したいときは必ず `$ZV_PLUGIN_DATA` の中に書いてください。**
ユーザーのプロジェクトを汚さずに済み、プラグインを消せば一緒に消えます。

## CLI から操作する

`zai` コマンドで、動いているエディタを外から操作できます。
プラグインからはもちろん、**エージェント自身に操作させることもできます**。

```bash
zai open src/main.rs --line 42     # ファイルを開く
zai notify "ビルドが通りました"       # 通知を出す
zai prompt "この関数をテストして"      # エージェントの入力欄へ差し込む
zai prompt "整形して" --submit       # 差し込んで送信まで行う
zai run "cargo test"               # ターミナルで実行
zai panel tasks "$(cat report.md)" # パネルを更新
zai status "デプロイ中"              # ステータスバーに表示
zai state                          # 現在の状態を JSON で出力

zai plugin list                    # プラグイン一覧
zai plugin new <名前>               # 雛形を作る
zai plugin enable <名前>            # 有効化
zai plugin disable <名前>           # 無効化
```

引数なしの `zai` や `zai .` は今までどおり GUI が起動します。
上に挙げたサブコマンド名を第 1 引数に置いたときだけ CLI として動きます。

CI やタスクランナーから `zai notify` を叩けば、
長い処理の完了をエディタ側で受け取る、といった使い方もできます。

### 覚えておくとよいこと: 背面のウィンドウは眠ります

macOS はウィンドウが前面に無いアプリの動作を止めます(実測で 10 秒間に
CPU 時間 0.01 秒程度)。そのためエディタが背面にあるときは、
**画面を読む操作と、指示を送る操作とで挙動が分かれます**。

| 種類 | コマンド | 背面のときの挙動 |
|---|---|---|
| 指示を送る | `open` `notify` `prompt` `run` `panel` `status` | **すぐ成功します**。指示は積まれ、エディタが次に動いたときに必ず適用されます |
| 状態を読む | `state` | 待っても値が返らないため、エラーになります。一度エディタを前面に出してから実行してください |

つまり `zai notify "ビルドが通りました"` のような使い方は、
エディタを触っていなくても取りこぼしません。
一方 `zai state` でその瞬間の状態を取りたいときだけ、エディタが起きている必要があります。

## テーマとスニペット

```toml
[[theme]]
label = "わたしの夜"
path = "themes/my-night.json"

[[snippet]]
language = "rust"
path = "snippets/rust.json"
```

- テーマは一般的なエディタ用カラーテーマ JSON 形式（コメント付き JSON も可）です。
- スニペットも一般的な形式で、`${1:既定値}` のタブストップ・`$0`・変数に対応します。
  prefix を打って Tab で展開されます。日本語を含んでいても崩れません。
- `language` を省略すると全言語で使えます。

## 有効・無効の切り替え

サイドバーの 🔌 タブでトグルできます。設定ファイルから切ることもできます。

```toml
[plugins]
disabled = ["usage-meter"]
```

無効にしたプラグインは、コマンド・フック・パネル・キーバインド・テーマ・
スニペットのすべてを登録しません。一覧には残るので、いつでも戻せます。

標準プラグインも無効にできます。

## 配布と受け取り

サイドバーの 🔌 タブに 3 つのボタンがあります。

- **➕ 新規作成** — 雛形一式を生成する
- **📤 エクスポート** — `<名前>-<バージョン>.zvplug` を書き出す
- **📦 インストール** — もらった `.zvplug` / `.zip` を選ぶだけ

`.zvplug` の実体はただの zip です。中身は `plugin.toml` を含むフォルダ 1 つ。
GitHub にそのまま置いても構いませんし、フォルダを直接
`~/.zaivern/plugins/` にコピーしても同じように動きます。

> インストールは同名のプラグインを上書きします。手を入れたものがあるなら
> エクスポートしてから受け取ってください。

## 標準プラグイン一覧

インストール直後から、以下がすべて有効な状態で入っています。
中身はただのシェルスクリプトなので、**読んで、真似して、書き換えて構いません**。

| プラグイン | できること |
|---|---|
| `worktrees` | 作業ツリーを分けて並列に走らせる。1 つの指示を複数のエージェントへ同時に渡し、結果を見比べて取り込む |
| `agent-compare` | 並列で走らせた結果を突き合わせ、変更量を並べて比較し、採用するものを選んで取り込む |
| `diff-review` | 差分の行にコメントを溜め、まとめてエージェントへ返す |
| `tasks` | 課題や変更依頼を一覧し、そこから作業ブランチを起こす。差分表示やコメント投稿も行う |
| `remote-host` | 別のマシン上で実行・同期・エージェント起動を行う |
| `element-capture` | 画面上の要素を選び、その構造・スタイル・切り抜き画像をプロンプトへ渡す |
| `usage-meter` | エージェントの利用状況をパネルに表示する |
| `quick-actions` | プロジェクトの種類を判別し、テスト・ビルド・整形をすぐ実行する |

自分の環境に合わないものは無効にして構いません。
**足りないものは、このガイドの手順で足してください。**

## 困ったとき

**コマンドが一覧に出てこない**
`plugin.toml` の書式エラーの可能性があります。🔌 タブでそのプラグインに ⚠ が
付いていれば理由が表示されます。フォルダ名と `name` が食い違っていないかも確認してください。

**実行しても何も起きない**
`output` が `silent` になっていないか確認してください。
まず `output = "notify"` にして、スクリプトの出力をそのまま目で見るのが近道です。

**エラー内容が読めない**
標準エラー出力はトーストに出ますが長いと切られます。
`run = "sh scripts/x.sh 2>&1"` として標準出力へ寄せ、`output = "new_tab"` にすると全文が読めます。

**アクションが効かない**
`output = "actions"` になっているか、JSON が **1 行 1 アクション** になっているかを確認してください。
整形された複数行の JSON は読めません。

**スクリプトが見つからない**
作業ディレクトリはワークスペースのルートです。
`"$ZV_PLUGIN_DIR/scripts/run.sh"` のように絶対パスで指定してください。

**変更が反映されない**
🔌 タブの ⟳ で再読み込みしてください。`plugin.toml` を書き換えたときは必須です。
